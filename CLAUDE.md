## Overview

**Cuponito** is a React-based coupon discovery platform built as a SPA (Single Page Application). It aggregates discount codes from partner stores, includes an admin panel for content management, and a blog for articles. The backend is Supabase (PostgreSQL + Auth), and the frontend is deployed on Vercel.

## Key Architecture Patterns

### Routing (TanStack Router)
- Routes are file-based: file name becomes the route pattern (e.g., `blog.$slug.tsx` → `/blog/:slug`)
- Route tree is auto-generated into `src/routeTree.gen.ts` via Vite plugin
- All route definitions must be in `src/routes/` directory
- Pages are separate from route files; routes import pages from `src/pages/`

### Data Fetching (TanStack Query)
- Custom hooks in `src/hooks/` wrap `useQuery` and `useMutation`
- Example: `useCoupons()` fetches from Supabase with 5-minute cache (staleTime)
- All hooks follow pattern: query key → Supabase query → error handling → return typed data
- Queries are cached and refetched based on stale times

### Database Integration (Supabase)
- Client initialized in `src/integrations/supabase/client.ts` with types from `types.ts`
- All queries go through the Supabase JS SDK
- Error handling required for every query (Supabase returns `{data, error}`)
- Auth state managed via Supabase (login/logout flow in admin pages)

### Component Pattern
- Each new component gets its own file in `src/components/`
- Components are presentational (props-driven, no direct queries)
- Data fetching happens in pages or custom hooks, passed as props to components
- Styling uses Tailwind utility classes + Shadcn components

### Dynamic Configuration (Zero-Code)
Per the plan in `plan.md`, all appearance and content must be database-driven:
- Store branding (colors, icons, gradients) fetched from `stores` table
- Admin dashboard allows editing store metadata
- SEO metadata (titles, descriptions) must come from database, not hardcoded
- No hardcoded URLs, IDs, or configuration values

## Important Development Rules

### ✅ DO
- Use TanStack Query hooks for all data fetching
- Import components from `@/components/` using path alias
- Fetch store styling and config from Supabase (never hardcode)
- Use the `cn()` utility from `@/lib/utils.ts` to merge Tailwind classes
- Add `rel="nofollow sponsored noopener noreferrer"` to external affiliate links
- Follow SOLID principles (single responsibility, dependency injection)
- Use async/await for async operations

### ❌ DON'T
- Edit `src/routeTree.gen.ts` directly (auto-generated by Vite plugin)
- Import directly from `node_modules` for UI components (use Shadcn wrappers)
- Hardcode store names, URLs, colors, or limits
- Create new routes outside of `src/routes/`
- Use class component syntax (prefer functional components)
- Import `react-router-dom` (using TanStack Router instead)

## Path Aliases

Use `@/` prefix for all imports from `src/`:
- `@/components/Header`
- `@/hooks/useCoupons`
- `@/lib/utils`
- `@/integrations/supabase/client`

## Supabase Integration

### Client Usage
```typescript
import { supabase } from '@/integrations/supabase/client';

// Type-safe queries using auto-generated types
const { data, error } = await supabase
  .from('coupons')
  .select('*')
  .eq('status', true);
```

### Custom Hook Pattern
```typescript
import { useQuery } from '@tanstack/react-query';

export function useMyData() {
  return useQuery({
    queryKey: ['myData'],
    queryFn: async () => {
      const { data, error } = await supabase...;
      if (error) throw error;
      return data || [];
    },
    staleTime: 5 * 60 * 1000, // 5 minutes
  });
}
```

### Auth Middleware
Auth state is managed via `src/integrations/supabase/auth-middleware.ts`. Admin pages check session before rendering.

## Common Tasks

### Add a New Page
1. Create component in `src/pages/MyPage.tsx`
2. Create route file `src/routes/mypage.tsx` that exports the page
3. Router rebuilds automatically; test at `/mypage`

### Add a Custom Hook for Data
1. Create `src/hooks/useMyData.ts`
2. Use `useQuery` from TanStack Query
3. Fetch from Supabase in the `queryFn`
4. Export typed data (use `Tables<'table_name'>` from Supabase types)

### Add a New Component
1. Create `src/components/MyComponent.tsx`
2. Import Shadcn or Tailwind as needed
3. Export as default
4. Import in page/parent with `import MyComponent from '@/components/MyComponent'`

### Update Admin Dashboard
1. Edit store metadata in Supabase directly
2. Pages fetch updated data on next load (React Query handles caching)

## Performance Considerations

- TanStack Query caches data with configurable `staleTime`
- `defaultPreload: 'intent'` in router preloads routes on hover
- Tailwind CSS is tree-shaken at build time
- SPA with client-side routing (no server-side rendering)
